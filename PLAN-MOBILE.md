# disbit → мобильное приложение за 7 дней

**Стратегия:** не переписываем ничего. Заворачиваем готовый веб (frontend/) в
нативную оболочку через **Capacitor** — наш HTML/CSS/JS работает как есть,
плюс получаем доступ к нативному: уведомления, вибрация, статус-бар.
React-переезд и реальные блокировки приложений — после релиза, не сейчас.

**Реалистичная цель недели:**

- ✅ Рабочее Android-приложение (APK/AAB) на своём телефоне — день 3
- ✅ Опубликовано в **RuStore** (модерация 1–3 дня, бесплатно) — конец недели
- ✅ Google Play: приложение загружено, идёт **закрытое тестирование**
- ⏳ Google Play **продакшн** — неделя 3–4 (правило Google: новым личным
  аккаунтам нужно 14 дней закрытого теста с ~12 тестерами — это не обойти)
- ⏳ iOS — недели 2–4 (нужен Mac или облачная сборка + Apple Developer $99/год)

---

## День 1 (пн) — Capacitor и первый запуск на телефоне

- Установить: Android Studio, JDK 17, Node LTS.
- В корне проекта: `npm init @capacitor/app`, скопировать `frontend/` в `www/`
  (или указать `webDir: "frontend"` в `capacitor.config.ts`).
- `npm i @capacitor/core @capacitor/android && npx cap add android`.
- **Шрифты локально**: скачать Onest и Unbounded (woff2) в `frontend/fonts/`,
  заменить Google Fonts `<link>` на `@font-face` — приложение обязано
  работать офлайн.
- `npx cap run android` → приложение на телефоне/эмуляторе.
- Проверить на реальном экране: safe-area, док, шторки, клавиатура.

**Итог дня:** disbit запускается как приложение на Android.

## День 2 (вт) — нативная оболочка

- `@capacitor/status-bar` — статус-бар в цвет темы (меняется с темой).
- `@capacitor/splash-screen` — сплэш с логотипом на фоне #0C1422.
- `@capacitor/haptics` — вибрация на отметке привычки, выполнении дня, штрафе.
- `@capacitor/app` — кнопка «назад» на Android: закрывает шторку → сворачивает.
- Иконка приложения + adaptive icon (генератор `@capacitor/assets` — из одного
  1024×1024 PNG делает все размеры и сплэши).
- Клавиатура: `@capacitor/keyboard`, чтобы шторки не прыгали.

**Итог дня:** ощущается как нативное приложение, не как сайт.

## День 3 (ср) — киллер-фича: напоминания

- `@capacitor/local-notifications`:
  - ежедневное напоминание в выбранное время: «Сегодня 3 привычки, под
    штрафом 250₽» (настройка времени — в Профиль → Настройки);
  - вечернее «Не всё закрыто — успей до полуночи» (если день не 100%);
  - перепланирование при изменении привычек.
- Автоитог при открытии уже работает — убедиться, что resume-событие
  приложения (`App.addListener('resume')`) перезапускает `settlePastDays()`
  при смене дня без перезапуска.
- Полный прогон на телефоне: создать привычки, отметки, минимум, календарь,
  ретро-отметки, темы, аватар с фото (камера/галерея через `<input type=file>`
  работает в WebView из коробки).

**Итог дня:** приложение готово функционально. Можно ставить друзьям.

## День 4 (чт) — бэкенд в интернет (аккаунты) 

- Захостить backend: **Railway / Render / VPS** (Express+SQLite работает без
  переделок; на Railway — volume под `db/disbit.db`). HTTPS из коробки.
- Во фронте: `const API` → адрес прод-сервера (конфиг в одном месте),
  CORS уже включён.
- Проверить в приложении: регистрация, вход, синхронизация между телефоном
  и вебом (disbit.vercel.app туда же переключить).
- ⚠️ Если день горит — аккаунты можно отложить: приложение полноценно
  работает офлайн на localStorage. Тогда день 4 уходит на полировку.

**Итог дня:** один аккаунт — на телефоне и в вебе.

## День 5 (пт) — витрина и обязательные бумаги

- **Политика конфиденциальности** — обязательна для обоих сторов. Страница
  на disbit.vercel.app/privacy (что храним: привычки, email; куда: свой
  сервер; без продажи данных).
- Онбординг: 3 экрана при первом запуске (идея штрафов → грейс-день и
  минимум → награды).
- Скриншоты для сторов: 4–6 штук с телефона (Сегодня, Календарь, Статистика,
  Профиль, тёмная+светлая тема). Плюс feature-баннер 1024×500 для Play.
- Тексты: короткое описание (80 зн.), полное (до 4000 зн.), ключевые слова.
- Прогон на 2–3 устройствах/эмуляторах (маленький экран, планшет).

**Итог дня:** всё для карточки приложения готово.

## День 6 (сб) — релизные сборки и кабинеты

- **Keystore** для подписи Android (`keytool -genkey ...`) — сохранить в
  надёжное место, потеря = невозможность обновлять приложение!
- Собрать подписанный **AAB** (`Build → Generate Signed Bundle`).
- Версионирование: `versionCode 1`, `versionName "0.5.0"`, `applicationId`
  вида `io.disbit.app` (или свой домен наоборот).
- Кабинеты:
  - **RuStore Console** (console.rustore.ru) — регистрация бесплатная,
    физлицо ок → загрузить AAB/APK, карточка, анкета возрастного рейтинга.
  - **Google Play Console** — $25 однократно → загрузить AAB во «Внутреннее
    тестирование», заполнить Data Safety, рейтинг, создать закрытый трек.

**Итог дня:** сборки загружены в оба стора.

## День 7 (вс) — отправка и запуск тестирования

- RuStore: отправить на модерацию (обычно 1–3 рабочих дня → публикация).
- Google Play: запустить **закрытое тестирование**, позвать ~12 тестеров
  (друзья/чат — им нужно принять приглашение и поставить приложение;
  тест должен идти 14 дней — таймер начинается сейчас).
- Буфер на правки по итогам прогонов.

**Итог недели:** приложение у тебя и тестеров; RuStore — на модерации
(публикация в начале недели 2); Play production — через ~2 недели теста.

## После недели

1. **Google Play production** — после 14 дней теста подать заявку (неделя 3–4).
2. **iOS** — `npx cap add ios`; сборка требует Mac (или Codemagic/GitHub
   Actions с macOS-раннером). Apple Developer $99/год, ревью 1–3 дня.
   Риск: guideline 4.2 «minimum functionality» для веб-обёрток — у нас уже
   есть нативные уведомления/haptics, добавить виджет/Live Activity позже.
3. **Реальные штрафы (деньги)** — юр. проработка + платёжка (ЮKassa/Stripe),
   политики сторов про реальные платежи (это не in-app purchase цифрового
   контента — вне комиссии стора, но нужна аккуратность в формулировках).
4. **Реальная блокировка приложений** — Android: UsageStats + Accessibility
   (жёсткая модерация Play), iOS: Screen Time API (FamilyControls,
   entitlement по заявке Apple). Отдельный большой этап.
5. React-переезд фронта (заготовка в `react/` уже есть).

## Главные риски

- **Play: 12 тестеров × 14 дней** — начать собирать тестеров уже в день 1.
- **Потеря keystore** = смерть приложения в Play — два бэкапа.
- **iOS без Mac** — закладывать облачную сборку, не тратить неделю на это.
- **WebView-производительность** на старых Android — blur ауры/стекла
  проверить на слабом устройстве, при лагах уменьшить blur через media query.
