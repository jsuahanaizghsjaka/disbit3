# disbit — бэкенд (каркас)

Пока это **заготовка**. Сервер работает, но хранит данные в памяти
(сбрасываются при перезапуске). Настоящая база данных подключится позже —
её структура уже описана в [`db/schema.sql`](db/schema.sql).

## Запуск

```bash
cd backend
npm install
npm start        # http://localhost:3000
# или npm run dev — авто-перезапуск при изменениях
```

Проверка: открой http://localhost:3000/api/health → должно вернуть `{ "ok": true }`.

## Эндпоинты (уже есть, на временном хранилище)

| Метод  | Путь               | Что делает              |
|--------|--------------------|-------------------------|
| GET    | `/api/habits`      | список привычек         |
| POST   | `/api/habits`      | создать привычку        |
| PUT    | `/api/habits/:id`  | обновить привычку       |
| DELETE | `/api/habits/:id`  | удалить привычку        |

## Что дальше (план)

1. **Подключить БД** — взять `db/schema.sql`, создать SQLite-базу,
   заменить массив в `routes/habits.js` на запросы к базе.
2. **Авторизация** — регистрация/вход, привязка привычек к `user_id`.
3. **Платежи** — Stripe/ЮKassa: сохранение карты и реальное списание ставки
   (таблица `charges`, статус меняется с `simulated` на `charged`).
4. **Подключить фронтенд** — заменить `localStorage` на запросы к этому API.
5. **ИИ-проверка выполнения** — отдельный сервис (по задумке, позже).

> Реальное списание денег и блокировка приложений в вебе невозможны —
> это задачи для нативного приложения + этого бэкенда. Сейчас всё симулируется.
