# disbit — бэкенд

Express + **SQLite** через встроенный модуль `node:sqlite` — никаких нативных
зависимостей. База лежит в файле `db/disbit.db` (создаётся автоматически из
[`db/schema.sql`](db/schema.sql) при первом запуске).

**Требование:** Node.js ≥ 22.13 (в нём `node:sqlite` доступен без флагов).

## Запуск

```bash
cd backend
npm install
npm start        # http://localhost:3000
# или npm run dev — авто-перезапуск при изменениях
```

Проверка: открой http://localhost:3000/api/health → должно вернуть `{ "ok": true }`.

Сервер также раздаёт фронтенд: открой **http://localhost:3000** — приложение
будет автоматически синхронизироваться с базой (при первом заходе локальные
данные из браузера заливаются на сервер).

## Эндпоинты

| Метод  | Путь                          | Что делает                                  |
|--------|-------------------------------|---------------------------------------------|
| GET    | `/api/health`                 | проверка, что сервер жив                    |
| GET    | `/api/habits`                 | список привычек с историей отметок          |
| POST   | `/api/habits`                 | создать привычку (можно с историей)         |
| PUT    | `/api/habits/:id`             | обновить привычку (прогресс не трогает)     |
| PUT    | `/api/habits/:id/day/:day`    | отметка за день `{ done, count }`           |
| DELETE | `/api/habits/:id`             | удалить привычку (отметки — каскадом)       |
| GET    | `/api/charges`                | журнал списаний (симуляция)                 |
| POST   | `/api/charges`                | записать списания (дубли за день игнорятся) |

## Что дальше (план)

1. **Авторизация** — регистрация/вход, привязка привычек к `user_id`
   (таблица `users` уже в схеме, пока всё живёт под user_id = 1).
2. **Платежи** — Stripe/ЮKassa: сохранение карты и реальное списание ставки
   (таблица `charges`, статус меняется с `simulated` на `charged`).
3. **ИИ-проверка выполнения** — отдельный сервис (по задумке, позже).
